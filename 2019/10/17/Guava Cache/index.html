Guava Cache

在内存中缓存数据，相比较于数据库和redis存储，访问内存中的数据会更加高效。其内部数据结构类似于ConcurrentHashMap，Segement数组+Entry链表数组。

Guava Cache

Maven

Maven仓库最新版本

什么时候使用

  • 愿意消耗内存空间来提升速度。
  • 预料到某些键会被多次查询。
  • 缓存中存放的数据总量不会超过内存容量。

构建缓存对象

通过使用CacheBuilder类构建缓存对象。CacheBuilder类采用builder设计模式，它的每个方法都返回CacheBuilder本身，直到build()方法被调用。

设置最大存储量

Guava Cache可以在构建缓存对象时指定缓存所能够存储的最大记录数量。当Cache中的记录数量达到最大值后再调用put()方法向其中添加对象，Guava会先从当前缓存的对象记录中选择删除掉，再将新的对象存储到Cache中。

设置过期时间

在构建Cache对象时，可以通过CacheBuilder类的expireAfterAccess()-指定对象多久没有被访问后过期和expireAfterWrite()-指定对象被写入到缓存多久后过期方法为缓存中的对象指定过期时间。过期的对象将会被缓存自动删除。

数据刷新

可以通过refreshAfterWrite()方法，超过refreshAfterWrite时间没有更新数据，则调用reload()方法刷新数据。

弱引用

可以通过weakKeys() 和weakValues()方法指定Cache只保存对缓存记录key和value的弱引用。当没有其他强引用指向key和value时，key和value对象就会被垃圾回收器回收。

显示清除

可以调用Cache的invalidateAll()-批量删除或invalidate()-删除一条记录，接收要删除记录的key作为参数方法显示删除Cache中的记录。

移除监听器

可以为Cache对象添加一个移除监听器，当有记录被删除时可以感知到这个事件。

自动加载

Cache的get()方法有两个参数，第一个参数是从Cache中获取记录的key，第二个参数是Callable对象。
当缓存中已经存在key对应的记录时，get()方法直接返回key对应的记录。如果缓存中不包含key对应的记录，Guava会启动一个线程执行Callable对象中的call()方法。call()方法的返回值会作为key对应的值被存储到缓存中，并且被get()方法返回。

从结果中可以看出，虽然是两个线程同时调用get方法，但只有一个get方法中的Callable会被执行(没有打印出load2)。Guava可以保证当有多个线程同时访问Cache中的一个key时，如果key对应的记录不存在，Guava只会启动一个线程执行get方法中Callable参数对应的任务加载数据存到缓存。当加载完数据后，任何线程中的get方法都会获取到key对应的值。

统计信息

可以对Cache的命中率、加载数据时间等信息进行统计。在构建Cache对象时，可以通过CacheBuilder的recordStats()方法开启统计信息的开关。调用Cache的stats()方法可以查看统计后的信息。

  • hitRate():缓存命中率
  • averageLoadPenalty():加载新值的平均时间，单位：纳秒
  • evictionCount():缓存项被回收的总数，不包括显示清除

CacheLoader

class

com.google.common.cache.CacheLoader

Computes or retrieves values, based on a key,for use in population a LoadingCache。
根据键计算或检索值，以用于填充LoadingCache。

LoadingCache

interface

com.google.common.cache.LoadingCache

A semi-persistent mapping from keys to values. Values are automatically loaded by the cache, and are stored in the cache until either evicted or manually invalidated.
从键到值的半持久性映射。值由缓存自动加载，并存储在缓存中，直到清除或手动失效。

  1. LoadingCache是Cache的子接口，相比较于Cache，当从LoadingCache中读取一个指定key的记录时，如果该记录不存在，则LoadingCache可以自动执行加载数据到缓存的操作。

  2. 使用CacheLoader实例化LoadingCache:

与构建Cache类型的对象类似，LoadingCache类型的对象也是通过CacheBuilder进行构建，不同的是，在调用CacheBuilder的build方法时，必须传递一个CacheLoader类型的参数，CacheLoader的load方法需要我们提供实现。当调用LoadingCache的get方法时，如果缓存不存在对应key的记录，则CacheLoader中的load方法会被自动调用从外存加载数据，load方法的返回值会作为key对应的value存储到LoadingCache中，并从get方法返回。

  • load()时，只会有一个线程去执行load()，其他线程会被阻塞，直到数据加载成功。

  • reload()时，只会有一个线程去执行reload()，其他线程会返回oldValue()。官网推荐重写reload()方法。
